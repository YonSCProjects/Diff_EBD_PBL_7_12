Tier 2 • Milestone 4 of 5

Add the button, pick its behaviour

Project 1 • Light Signals
In this milestone you add a push-button to your breadboard and decide what the button does. There are three options. Pick the one that fits the pattern you are building. Then use Claude Code to change your sketch so the button actually does it.

Step 1 — Wire the button

Same wiring as Tier 1 Milestone 7: button straddles the centre gap, pin A to 5 V, pin B to Arduino pin 2, and a 10 kΩ pull-down resistor from pin 2 to GND. See reference card R1 Wiring 3 for the full diagram.

Pull-down reminder The 10 kΩ resistor goes between pin 2 and GND. Not between 5 V and GND. Getting this wrong causes the Arduino to reset. Check the reference card R1 if you are unsure.

Step 2 — Pick ONE button behaviour

Behaviour A — Toggle
Pressing the button once starts the pattern. Pressing it again stops the pattern. Press once more: starts again.
Behaviour B — Cycle
Pressing the button cycles through different patterns. Press once for alternating, press again for chasing, press again for breathing, press again to go back to alternating. (You will need all three LEDs wired for this one.)
Behaviour C — Speed control
Holding the button down makes the pattern run faster. Releasing the button brings it back to normal speed.

Step 3 — Use Claude Code Level 2 to make the button do your chosen behaviour

Same (a)(b)(c) discipline as Milestone 3. Describe the behaviour you want first. Then ask Claude Code to help you modify the sketch.

(a) What I want to happen: [describe your chosen behaviour — A, B, or C — in your own words] (b) What is currently happening: [describe what your sketch does right now, before the button works] (c) What I have tried: [what you have already looked at or changed]
What you should see The button changes your pattern in the way you picked. Behaviour A: pressing starts and stops. Behaviour B: pressing cycles through patterns. Behaviour C: pressing makes it faster. You can show the behaviour to the teacher and say "this is what I built."
Done when
  • The button is wired with the 10 kΩ pull-down in the correct position
  • You chose a button behaviour (A, B, or C)
  • Your sketch has been modified via Claude Code Level 2 to do the chosen behaviour
  • The button actually does what you chose
Stuck? Button does nothing? First check the pull-down position on R1. If wiring is correct but the button still does not work, the sketch probably needs pinMode(BUTTON, INPUT); in setup() and a digitalRead() in the loop. Ask Claude Code to check your code for that. If Claude's answer does not help, call the teacher.
