Tier 3 • Project Planner

Design your own Project 1

Project 1 • Light Signals
Tier 3 is for students who want to design their own version of Project 1 from scratch, rather than follow the Tier 1 or Tier 2 milestones. At Tier 3 you pick a real use case — a mood light, a bike signal, a two-player reaction game, a Morse-code sender, anything that uses LEDs and maybe a button — and you build it. There are no numbered milestones. There is this one planner. You fill it in, you build it, you show it.

Phase 1 — Plan

Example: "A mood light for my desk. Three LEDs. Button picks the colour mood — calm, energetic, or neutral."
Knowing who it is for helps you make design choices. "A bike signal for me" is different from "a visual reminder for my little brother to brush his teeth."
For most Tier 3 projects you will use what is in your parts tray: 2–4 LEDs, 2–4 resistors, a button, jumper wires. If you need something unusual, ask the teacher before you start.
This is the most important field. Be as specific as you can. "The LEDs do something" is not enough. "When I press the button, the LEDs cycle through red → yellow → green → off, one press each, and stay at whatever colour you stopped on" is enough to start coding.

Phase 2 — Build

Use reference card R1 as a base. Your wiring is probably a variation on Wiring 1, 2, or 3. Draw it on paper before you wire it — even a rough sketch helps.

Phase 3 — Code (Claude Code Level 3)

At Tier 3 you use Claude Code in free-dialogue mode (Channel A Level 3). You describe what you are trying to build, Claude Code helps you write the sketch, and you iterate together until it works. The same (a)(b)(c) discipline still applies — describe the problem before asking for a solution.

Something like: "I am building a mood light for my desk. Three LEDs on pins 9, 10, 11 with 220 Ω resistors, and a push-button on pin 2 with a 10 kΩ pull-down. I want pressing the button to cycle through colour combinations. Can you help me write the Arduino sketch?"

Phase 4 — Test

If no: what is different? Write it down, go back to Claude Code with a new (a)(b)(c), iterate.
Intermittent bugs are usually either wiring (a loose jumper) or timing (a delay that is too short or too long).

Phase 5 — Show

  • When you are happy with it (or when you have decided it is as good as it is going to be today), call the teacher.
  • Show your project. Explain in one or two sentences what it does and why you built it.
  • If you want, take a photo of the finished project and save it to your Project 1 folder on Google Drive.
🎉 You just completed Tier 3 of Project 1! You designed, built, and programmed your own version of Project 1 from scratch. This is the kind of work the whole Arduino program is building toward.
Stuck at any phase? Tier 3 is open-ended and that is its point. Being stuck is part of designing your own project. Use Claude Code to think through the problem, and call the teacher if you have been stuck for more than 10 minutes on the same thing. A teacher conversation at Tier 3 is a design conversation, not a troubleshooting one — tell them what you are trying to build and what is not working.
