Teacher Setup Checklist

Project 1 • Light Signals • Pre-session checklist • Target time: ≤ 15 minutes

Complete this checklist before the first student arrives. Items marked [first cohort only] are one-time setup; items with no tag are per-session. [Verification: Principle 9 — teacher sustainability as a first-class design constraint. Pre-session prep is intentionally bounded to ≤15 minutes per the sustainability target in Arduino_PBL_Program.md §7.2.]

Part 1 — Hardware setup (per station)

  1. Lay out each workstation with: Arduino Uno, USB-B cable plugged into the PC, breadboard, one LED + one 220 Ω resistor in a small parts tray, a set of ~15 jumper wires. Do not pre-wire anything. The student wires from scratch — that is the point of Milestones 3 onward.
    [Verification: Principle 3 (physical-first, concrete-before-abstract) — Hwang & Taylor (2016) CRA sequence. The student's own hands on the wire is the intervention, not the finished circuit.]
  2. Place the 5 reference cards at each station: R1 Wiring, R2 Stuck Protocol, R3 Claude Code Prompts, R4 Safety, R5 Sketch Index. Laminated, clipped or loose, within the student's immediate reach.
    [Verification: Principle 2 (task-card-is-the-boss) — Rosen, Boyle, Cariss & Forchelli (2014) on external cognitive scaffolds. The reference cards are the student's first line of help, not the teacher.]

Part 2 — Arduino IDE setup (per PC)

  1. Open the Arduino IDE on each PC. Confirm the board type (Tools → Board → Arduino Uno) and the correct COM port (Tools → Port → COMx). Do a test compile-and-upload of 01_blink_L_fast.ino to confirm the IDE is talking to the board. This is the highest-risk setup failure; check it before students arrive.

Part 3 — Shared Workshop Drive and Claude Code setup (per PC)

  1. Confirm Google Drive for Desktop is running and logged into the shared robotics-workshop Google account, with the account's My Drive mounted at G:\My Drive\. If Drive for Desktop is not running, start it from the Windows Start menu and wait for the Drive to finish syncing before students arrive.
  2. Open File Explorer and navigate to G:\My Drive\Arduino_Projects\. This folder holds one subfolder per student nickname for the whole robotics program. For each student in today's session:
    • If the student has worked in the workshop before, their nickname subfolder already exists. Confirm it is visible.
    • If this is the student's first time ever in the workshop, their nickname subfolder does not exist yet and the student has not picked a nickname. Both of those happen at Milestone 1 in the together-ritual — you and the student pick a nickname together and create the folder together. Do NOT pre-create a folder with a guess at the nickname.
    [Verification: Principle 5 (structured autonomy) + Principle 8 (teacher-student relationship as the multiplier) — Sciacca (2025) building-trusting-relationships sub-theme. The nickname and folder creation are the student's first Principle 5 moment, operationalised via a one-minute together-ritual.]
  3. Do NOT pre-create the Project_1_Light_Signals subfolder for any student. That folder is created in the together-ritual at Milestone 1. The student watches the folder come into existence. Creating it ahead of time removes the ritual.
  4. Open Claude Code on the workstation but do not launch it into a working directory yet. Claude Code is launched into the student's Project 1 folder during Milestone 1, after the folder is created together. A Claude Code session that was left open from a previous student with a different working directory should be closed and restarted fresh for the new student.

Part 4 — Per-student sketch preparation

  1. For students whose Project 1 folder already exists from a previous session: confirm the ino_files subfolder is present inside their Project 1 folder with all seven sketch subfolders. If it is missing, copy the entire ino_files directory from the repository's Arduino_Projects\Project_1_Light_Signals\ino_files\ into the student's Project 1 folder before the session. Session-flow decision: Option A — all sketches pre-copied at session start, not added per milestone. This reduces teacher rotation burden during the session.
  2. For students whose Project 1 folder does not yet exist: nothing to copy yet — the folder is created at Milestone 1. Immediately after Milestone 1 the teacher copies the ino_files directory into the newly-created folder, while the student is plugging in the Arduino and looking at the factory-blink L LED (this is a 30-second copy operation that the teacher does during the student's natural hardware-looking pause).

Part 5 — Tracking and session-readiness

  1. Review the tracking sheet for the expected students today (the §5.8 sheet from the master document). Note any students whose last-session row had a red-card incident — those students need an intentional relational check-in during Arrival. Have your private nickname-to-real-name mapping sheet ready to record a new nickname if this is a first-session student.
    [Verification: Principle 8 — teacher-student relationship as the multiplier. Pre-session awareness of each student's recent state shapes the Arrival phase.]
  2. Confirm the cool-down corner is set up at the back of the workshop (§5.6 of the master document). Not Project 1-specific but always part of session setup.
Pre-session time target: ≤ 15 minutes. If this checklist takes longer than 15 minutes per session consistently (more than two weeks in a row), flag it in the Weekly Data Collector — it is a signal that the checklist needs simplification, not that you need to work harder.
