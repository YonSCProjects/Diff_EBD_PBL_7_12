איך משתמשים במדריך הזה

המדריך הזה מכיל את כל פרויקט 1 בעמוד אחד. זה אותו תוכן של כרטיסיות הניווט המודפסות שעל השולחן — רק על המסך במקום על נייר. אפשר לקרוא את המדריך, או את הכרטיסיות המודפסות, או להשתמש בשניהם יחד. הם אומרים אותו דבר.

איך פותחים קובצי קוד ב-Arduino IDE. בשלב 1, המורה יעזור לכם להגדיר את מיקום ה-Sketchbook של Arduino IDE לתיקיית פרויקט 1 שלכם (File → Preferences → Sketchbook location). אחרי זה, כל קובצי הקוד שלכם מופיעים תחת File → Sketchbook — לחיצה אחת לפתיחה. ככה פותחים כל קוד בסדנה.

כפתורי צפייה בקוד. כשרואים כפתור כחול כמו 📎 צפייה בקוד: filename.ino הוא מציג את הקוד בדפדפן לקריאה לצד ההוראות. כדי באמת להעלות את הקוד, פותחים אותו ב-Arduino IDE דרך File → Sketchbook.

בוחרים גרסה — גרסה 1 (בנייה מודרכת, 8 שלבים), גרסה 2 (עיצוב מודרך, 5 שלבים), או גרסה 3 (עיצוב פתוח, מתכנן פרויקט). אם זו הפעם הראשונה בסדנה, מתחילים בגרסה 1.

  digitalWrite(LED_BUILTIN, LOW);

}

בשלב זה מחווטים לד על הברדבורד בפעם הראשונה. חומרה בלבד — בלי העלאת קוד.

📋מה עושים

  • בוחרים לד אחד ממגש החלקים. כל צבע בסדר.
  • שמים לב: הרגל הארוכה היא חיובית (+), הרגל הקצרה היא שלילית (−).
  • בוחרים נגד של 220 אוהם (פסי צבע: אדום · אדום · חום).

  • הרגל הקצרה (−) ← חוט ג'אמפר ← GND של הארדואינו.

03_blink_alternating.ino// Blinks two external LEDs in alternation.

const int LED_1 = 9;
const int LED_2 = 10;

void setup() {
  pinMode(LED_1, OUTPUT);
  pinMode(LED_2, OUTPUT);
}

void loop() {
  digitalWrite(LED_1, HIGH);
  digitalWrite(LED_2, LOW);
  delay(500);

  digitalWrite(LED_1, LOW);
  digitalWrite(LED_2, HIGH);
  delay(500);
}

זה שלב החיווט הקשה ביותר בפרויקט 1. מוסיפים כפתור לחיצה וגם נגד מיוחד בשם נגד הורדה. חשוב מאוד להכניס את נגד ההורדה במקום הנכון — אם שמים אותו במקום הלא נכון, הארדואינו יאתחל את עצמו. קוראים בזהירות.

📋מה עושים

  • בוחרים כפתור לחיצה. יש לו ארבע רגליים.
  • מכניסים אותו לרוחב החריץ המרכזי של הברדבורד — ארבע רגליים בארבע שורות שונות.
  • בוחרים נגד של 10 קילו-אוהם (פסי צבע: חום · שחור · כתום). שונה מהנגד של 220 אוהם של הלד.
  • רגל A של הכפתור ← 5 V של הארדואינו.
  • רגל B של הכפתור ← רגל 2 של הארדואינו.
  • מרגל B, גם דרך הנגד של 10 קילו-אוהם ← GND. זה נגד ההורדה.

🔌תרשים החיווט

Arduino 5 V ──────────────┬─── Button pin A
                          │
                          ▼
                      Button pin B ────┬─── Arduino pin 2
                                       │
                                       └─[10 kΩ]── Arduino GND
⚠️הטעות הנפוצה ביותר נגד ההורדה של 10 קילו-אוהם הולך בין רגל 2 ל-GND. לא בין 5 V ל-GND. אם שמים אותו בין 5 V ל-GND, הלחיצה על הכפתור יוצרת קצר והארדואינו מתאתחל. בודקים באצבע: מרגל 2 ל-GND? בטוח. מ-5 V ל-GND? קודם כל מתקנים את זה.
👀מה אמורים לראות הכפתור על הברדבורד עם נגד ההורדה בין רגל 2 ל-GND. שני הלדים מקודם עדיין מהבהבים. עדיין לא קורה שום דבר חדש כשלוחצים על הכפתור — הקוד הנוכחי לא יודע על הכפתור. שלב 8 משנה את זה.

💻הקוד שמעלים

04_button_control.ino// A push-button controls which of two LEDs is on.

const int LED_1 = 9;
const int LED_2 = 10;
const int BUTTON = 2;

void setup() {
  pinMode(LED_1, OUTPUT);
  pinMode(LED_2, OUTPUT);
  pinMode(BUTTON, INPUT);
}

void loop() {
  int buttonState = digitalRead(BUTTON);

  if (buttonState == HIGH) {
    // Button is pressed: LED 2 on, LED 1 off.
    digitalWrite(LED_1, LOW);
    digitalWrite(LED_2, HIGH);
  } else {
    // Button not pressed: LED 1 on, LED 2 off.
    digitalWrite(LED_1, HIGH);
    digitalWrite(LED_2, LOW);
  }
}

שמים לב ל-if ... else. הארדואינו קורא את הכפתור. אם הכפתור לחוץ (HIGH), מאיר לד אחד. אם לא לחוץ (LOW), מאיר את השני. קלט ← החלטה ← פלט. זו התבנית הבסיסית של כל פרויקט אינטראקטיבי בארדואינו.

גרסה 3 היא לתלמידים שרוצים לעצב גרסה משלהם של פרויקט 1 מאפס. בוחרים רעיון שימושי אמיתי, בונים אותו, ומראים אותו. אין שלבים ממוספרות — רק חמישה שלבים במתכנן הזה.
בגרסה המודפסת של המתכנן יש שדות למילוי בעט. הגרסה הזו על המסך מראה את אותן שאלות — כותבים את התשובות על נייר, בקלוד קוד, או בקובץ טקסט נפרד.

📋שלב 1 — מתכננים

מה הפרויקט שלכם הולך להיות? (משפט או שניים)

דוגמה: "מנורת מצב רוח לשולחן שלי. שלושה לדים. כפתור בוחר את מצב הרוח של הצבע — רגוע, אנרגטי, או ניטרלי."

למי זה מיועד? (לעצמכם, לבן משפחה, לבית הספר, לשימוש מסוים)

לדעת למי זה מיועד עוזר להחליט החלטות עיצוב.

איזו חומרה צריך? (לדים, נגדים, כפתור, חוטי ג'אמפר — כמה מכל אחד)

ברוב פרויקטי גרסה 3: 2–4 לדים, 2–4 נגדים, כפתור, חוטי ג'אמפר. שואלים את המורה אם צריך משהו לא רגיל.

מה ההתנהגות? (ספציפי)

"הלדים עושים משהו" לא מספיק. "כשאני לוחץ על הכפתור, הלדים עוברים בין אדום → צהוב → ירוק → כבוי, לחיצה אחת כל פעם, ונשארים על הצבע שעצרתם בו" מספיק כדי להתחיל לקודד.

🔌שלב 2 — בונים

  • מציירים את החיווט על נייר קודם.
  • מחווטים על הברדבורד. משתמשים ב-R1 כבסיס.
  • מצלמים תמונה של החיווט הגמור (אופציונלי).

💻שלב 3 — קוד (ערוץ A — גרסה 3)

משתמשים בקלוד קוד במצב דיאלוג חופשי. מתארים מה אנחנו מנסים לבנות. משפרים יחד. אותה משמעת של (א)(ב)(ג) — מתארים את הבעיה לפני שמבקשים פתרון.

טיוטת הפנייה הראשונה (כותבים אותה לפני ששולחים):

אני בונה [תיאור הפרויקט שלכם] עבור [למי זה מיועד].
יש לי [רשימת החומרה] מחווטת על [אילו רגליים].
אני רוצה שההתנהגות תהיה [ספציפי].
תוכל לעזור לי לכתוב את הקוד בארדואינו?

📋שלב 4 — בודקים

  • מעלים ומסתכלים.
  • זה עושה מה שרציתם? אם לא, מה שונה?
  • חוזרים לקלוד קוד עם (א)(ב)(ג) חדשים.
  • משפרים עד שזה עובד — או עד שמחליטים שהדרך שזה יצא בפועל טובה יותר.

📋שלב 5 — מראים

  • קוראים למורה כשמרוצים.
  • מראים את הפרויקט. מסבירים מה זה עושה ולמה בניתם.
