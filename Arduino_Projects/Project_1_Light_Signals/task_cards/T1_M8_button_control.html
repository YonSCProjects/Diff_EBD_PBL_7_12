Upload the button-controlled sketch

Project 1 • Light Signals
In this milestone you upload code that uses the button: pressing it switches which LED is on. Your first interactive Arduino project — input (the button) changes output (the LEDs).

What to do

  • Open the file 04_button_control.ino in the Arduino IDE.It is in your Project 1 folder under ino_files/04_button_control/.
  • Click the Upload button.
  • Wait for "Done uploading" in green.
  • Look at your breadboard. One LED should be on, the other off. (This is the "button not pressed" state.)
  • Press and hold the button. The LEDs should swap — the one that was on turns off, and the one that was off turns on.
  • Release the button. The LEDs swap back.
  • Press and release a few times. The button should switch which LED is lit every time, reliably.

The sketch you are uploading

04_button_control.ino// A push-button controls which of two LEDs is on.

const int LED_1 = 9;
const int LED_2 = 10;
const int BUTTON = 2;

void setup() {
  pinMode(LED_1, OUTPUT);
  pinMode(LED_2, OUTPUT);
  pinMode(BUTTON, INPUT);
}

void loop() {
  int buttonState = digitalRead(BUTTON);

  if (buttonState == HIGH) {
    // Button is pressed: LED 2 on, LED 1 off.
    digitalWrite(LED_1, LOW);
    digitalWrite(LED_2, HIGH);
  } else {
    // Button not pressed: LED 1 on, LED 2 off.
    digitalWrite(LED_1, HIGH);
    digitalWrite(LED_2, LOW);
  }
}

Notice the if ... else in the loop. That is how the Arduino makes a decision. It reads the button, and depending on whether the button is pressed (HIGH) or not (LOW), it turns on one LED or the other. This is the fundamental pattern for every interactive Arduino project: read input → decide → write output.

What you should see With the button NOT pressed: LED 1 (on pin 9) is on, LED 2 (on pin 10) is off. When you press and hold the button: LED 1 turns off and LED 2 turns on. Releasing goes back to LED 1 on. The switch happens instantly when you press and again when you release.

Stuck? Button does nothing? The most common cause is the 10 kΩ pull-down resistor being in the wrong place. The pull-down must be between pin 2 and GND, not between 5 V and GND. If the button still does nothing after you verify the pull-down position, call the teacher.
