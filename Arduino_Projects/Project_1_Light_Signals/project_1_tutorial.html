Tier 1 • Milestone 3 of 8

Wire your first external LED

Project 1 • Light Signals
You are going to wire an LED on the breadboard for the first time. Hardware only — no sketch upload in this milestone.

What to do

  • Pick one LED from your parts tray. Any colour is fine.
  • Notice: the long leg is positive (+), the short leg is negative (−).
  • Pick a 220 Ω resistor (colour bands: red · red · brown).
  • Plant the LED across two rows of the breadboard — legs in different rows.
  • Long leg (+) → 220 Ω resistor → Arduino pin 9.
  • Short leg (−) → jumper wire → Arduino GND.

Wiring diagram

Arduino pin 9  ───[220 Ω]───┬─── LED long leg
                           │
                           ▼
                      LED short leg ─── Arduino GND

For a full-sized diagram with all four Project 1 wiring states, see R1 Wiring Reference.

What you should see The LED is in the breadboard, long leg connects through the 220 Ω resistor to pin 9, short leg connects to GND. The LED is not lit yet — that comes in Milestone 4 when you upload the sketch.
Done when
  • The LED is planted with legs in different rows
  • Long leg → 220 Ω → pin 9
  • Short leg → GND
Stuck? Check R1 Wiring Reference first — it shows exactly how the LED should look. If you are still unsure, call the teacher.

Project 1 • Light Signals
This is the most important milestone in Project 1. You wired the LED. Now you upload a sketch that makes it blink. The LED on your breadboard blinks because of code you uploaded. This is the moment code makes physical things happen.

What to do

View code: 02_blink_external.ino ino_files/02_blink_external/02_blink_external.ino
  • Open the sketch in the Arduino IDE (via File → Sketchbook).
  • Click the Upload button in the Arduino IDE.

The sketch you are uploading

const int LED_PIN = 9;

void setup() {
  pinMode(LED_PIN, OUTPUT);
}

void loop() {
  digitalWrite(LED_PIN, HIGH);
  delay(500);
  digitalWrite(LED_PIN, LOW);
  delay(500);
}
What you should see Your external LED on the breadboard blinks about once per second. Your wiring came alive because of the code you uploaded.
Done when
  • Your external LED is blinking once per second
  • "Done uploading" appeared in green
Stuck? LED not blinking?
  1. Flip the LED. Long leg to resistor, short leg to GND. About half the time the LED is backwards.
  2. Check the resistor is between pin 9 and the LED's long leg — not just stuck in the breadboard.
  3. Look for the green "Done uploading" message. If missing, upload failed.
  4. Still stuck? Call the teacher. See R1 Wiring Reference.

Upload the alternating-blink sketch

Project 1 • Light Signals
Now the Arduino controls two things at once. When one LED is on, the other is off. First time you see the Arduino do more than one thing in the same sketch.

What to do

View code: 03_blink_alternating.ino ino_files/03_blink_alternating/03_blink_alternating.ino
  • Open the sketch in the Arduino IDE (via File → Sketchbook).
  • Click Upload.
  • Wait for "Done uploading".
  • Watch your two LEDs. They should alternate — when LED 1 is on, LED 2 is off, then swap.

The sketch you are uploading

03_blink_alternating.ino// Blinks two external LEDs in alternation.

const int LED_1 = 9;
const int LED_2 = 10;

void setup() {
  pinMode(LED_1, OUTPUT);
  pinMode(LED_2, OUTPUT);
}

void loop() {
  digitalWrite(LED_1, HIGH);
  digitalWrite(LED_2, LOW);
  delay(500);

  digitalWrite(LED_1, LOW);
  digitalWrite(LED_2, HIGH);
  delay(500);
}
What you should see Two LEDs alternating — LED 1 on / LED 2 off / then swap / every half second.
Done when
  • Both LEDs are blinking in alternation
  • "Done uploading" appeared in green
Stuck? Both LEDs always on: one is probably wired to 5 V instead of a digital pin. Only one LED works: the other is wired backwards, or its resistor is not connected. See R1 Wiring 2.

Tier 1 • Milestone 7 of 8

Wire the push-button

Project 1 • Light Signals
Hardest wiring step of Project 1. You add a push-button AND a special resistor called a pull-down. Getting the pull-down in the right place is important — if you put it in the wrong place the Arduino will reset. Read carefully.

What to do

  • Pick a tactile push-button. It has four pins.
  • Plant it across the centre gap of the breadboard — four pins in four different rows.
  • Pick a 10 kΩ resistor (colour bands: brown · black · orange). This is different from the 220 Ω LED resistor.
  • Button pin A → Arduino 5 V.
  • Button pin B → Arduino pin 2.
  • From pin B, also through the 10 kΩ resistor → GND. This is the pull-down.

Wiring diagram

Arduino 5 V ──────────────┬─── Button pin A
                          │
                          ▼
                      Button pin B ────┬─── Arduino pin 2
                                       │
                                       └─[10 kΩ]── Arduino GND
Most common mistake The 10 kΩ pull-down goes between pin 2 and GND. NOT between 5 V and GND. If you put it between 5 V and GND, pressing the button creates a short circuit and the Arduino resets. Check with your finger: does the resistor go from pin 2 to GND? Good. From 5 V to GND? Move it.
What you should see The button is on the breadboard with the pull-down resistor between pin 2 and GND. The two LEDs from before are still blinking. Pressing the button does nothing yet — the current sketch doesn't know about the button. Milestone 8 changes that.
Done when
  • Button straddles the centre gap, pins in four different rows
  • Button pin A → 5 V
  • Button pin B → pin 2 AND → 10 kΩ → GND
  • The two LEDs from before are still alternating
Stuck? This is the hardest wiring step. The teacher expects to help here — call them. Before calling, check R1 Wiring 3 for the pull-down position. If the Arduino keeps resetting, you have the pull-down on the wrong side — unplug USB, fix it, plug back in.

Tier 1 • Milestone 8 of 8 • Final milestone

Upload the button-controlled sketch

Project 1 • Light Signals
Last milestone of Project 1. You wired the button in Milestone 7. Now you upload a sketch that uses the button to switch which LED is on. Pressing the button swaps which LED is lit. This is your first interactive Arduino project.

What to do

View code: 04_button_control.ino ino_files/04_button_control/04_button_control.ino
  • Open the sketch in the Arduino IDE (via File → Sketchbook).
  • Click Upload.
  • Wait for "Done uploading".
  • Look at your breadboard. One LED is on, the other is off.
  • Press and hold the button. The LEDs swap — the one that was on turns off, the one that was off turns on.
  • Release the button. The LEDs swap back.
  • Press and release a few times. The button should reliably switch the LEDs.

The sketch you are uploading

04_button_control.ino// A push-button controls which of two LEDs is on.

const int LED_1 = 9;
const int LED_2 = 10;
const int BUTTON = 2;

void setup() {
  pinMode(LED_1, OUTPUT);
  pinMode(LED_2, OUTPUT);
  pinMode(BUTTON, INPUT);
}

void loop() {
  int buttonState = digitalRead(BUTTON);

  if (buttonState == HIGH) {
    // Button is pressed: LED 2 on, LED 1 off.
    digitalWrite(LED_1, LOW);
    digitalWrite(LED_2, HIGH);
  } else {
    // Button not pressed: LED 1 on, LED 2 off.
    digitalWrite(LED_1, HIGH);
    digitalWrite(LED_2, LOW);
  }
}

Notice the if ... else. The Arduino reads the button. If the button is pressed (HIGH), it turns one LED on. If not pressed (LOW), it turns the other LED on. Read input → decide → write output. This is the fundamental pattern for every interactive Arduino project.

What you should see Button NOT pressed: LED 1 (pin 9) on, LED 2 (pin 10) off. Button pressed: LED 1 off, LED 2 on. The switch happens instantly and reliably.
Done when
  • Pressing the button switches which LED is lit
  • Releasing the button switches it back
  • You have completed Project 1
🎉 You just completed Project 1! You wired a breadboard from scratch. You uploaded four sketches. You built an interactive circuit with LEDs and a button. Ask the teacher for a photo of your finished build. Real electrical engineering.
Stuck? Button does nothing? Most common cause: the 10 kΩ pull-down resistor is on the wrong side. Check R1 Wiring 3. The resistor must be between pin 2 and GND, not between 5 V and GND. If the wiring is correct and the button still does nothing, call the teacher.

Add the button, pick its behaviour

Add a push-button and decide what it does. Then use Claude Code Level 2 to make your sketch actually do that behaviour.

Step 1 — Wire the button

Same wiring as Tier 1 Milestone 7: button across the centre gap, pin A to 5 V, pin B to Arduino pin 2, and a 10 kΩ pull-down resistor from pin 2 to GND. See R1 Wiring 3.

Pull-down reminder The 10 kΩ resistor goes between pin 2 and GND. Not between 5 V and GND. Getting this wrong causes the Arduino to reset.

Step 2 — Pick ONE button behaviour

Behaviour A — Toggle
Pressing the button starts the pattern. Pressing again stops it. Press again: starts.
Behaviour B — Cycle
Pressing the button cycles through patterns. Alternating → chasing → breathing → back to alternating. (Needs all three LEDs wired.)
Behaviour C — Speed control
Holding the button makes the pattern run faster. Releasing returns it to normal.

Step 3 — Use Claude Code Level 2 to implement your choice

Same (a)(b)(c) discipline as Milestone 3:

(a) What I want to happen:
[your chosen behaviour — A, B, or C — described in your own words]

(b) What is currently happening:
[what the sketch does right now, before the button works]

(c) What I have tried:
[what you already looked at or changed]

Tier 3 • Project Planner

Design your own Project 1

Project 1 • Light Signals
Tier 3 is for students who want to design their own version of Project 1 from scratch. Pick a real use case, build it, show it. No numbered milestones — just five phases on this planner.
The printed version of this planner has fill-in-the-blank fields you can write in with a pen. This on-screen version shows the same questions — write your answers on paper, in Claude Code, or in a separate text file.

Phase 1 — Plan

What is your project going to be? (one or two sentences)

Example: "A mood light for my desk. Three LEDs. A button picks the colour mood — calm, energetic, or neutral."

Who is it for? (yourself, a family member, the school, a specific use)

Knowing who it is for helps you make design choices.

What hardware do you need? (LEDs, resistors, button, jumper wires — how many)

For most Tier 3 projects: 2–4 LEDs, 2–4 resistors, a button, jumper wires. Ask the teacher if you need something unusual.

What is the behaviour? (be specific)

"The LEDs do something" is not enough. "When I press the button, the LEDs cycle through red → yellow → green → off, one press each, and stay at whatever colour you stopped on" is enough to start coding.

Phase 2 — Build

  • Sketch your wiring on paper first.
  • Wire it on the breadboard. Use R1 as a base.
  • Take a photo of your finished wiring (optional).

Phase 3 — Code (Claude Code Level 3)

Use Claude Code in free-dialogue mode. Describe what you are trying to build. Iterate together. Same (a)(b)(c) discipline — describe the problem before asking for a solution.

First prompt draft (write it before you send):

I am building [your project description] for [who it is for].
I have [list of hardware] wired on [which pins].
I want the behaviour to be [be specific].
Can you help me write the Arduino sketch?

Phase 4 — Test

  • Upload and watch.
  • Does it do what you wanted? If not, what is different?
  • Go back to Claude Code with a new (a)(b)(c).
  • Iterate until it works — or until you decide the way it actually came out is better.

Phase 5 — Show

  • Call the teacher when you are happy.
  • Show your project. Explain what it does and why you built it.
  • Save a photo to your Project 1 folder on Google Drive (optional).
🎉 You completed Tier 3 of Project 1! You designed, built, and programmed your own version of Project 1 from scratch. This is the kind of work the whole Arduino program is building toward.
Stuck at any phase? Tier 3 is open-ended and that is its point. Being stuck is part of designing your own project. Use Claude Code to think through the problem. Call the teacher if you have been stuck for more than 10 minutes. A Tier 3 teacher conversation is a design conversation, not a troubleshooting one — tell them what you are trying to build and what is not working.
